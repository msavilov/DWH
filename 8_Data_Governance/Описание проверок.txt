Я остановился на двух типах проверок:
1) Поиск дубликатов при совпадении всех значений, кроме id. Выполняется с помощью Unique rows и установкой влажка Redirect duplicate row, которая помечает все дубликаты как ошибки.
2) Поиск значений NULL в данных. Я выбрал только значение Цены товара, но можно проверить сразу и другие поля в рамках одного шага. Выполняется с помощью Filter rows и выбором функции is not null. Записи с null помечаются как ошибки

Далее все записи, помеченные как ошибки, записываются с помощью Table output в специальную таблицу в БД, для последующего ручного анализа
